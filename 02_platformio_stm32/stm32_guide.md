# STM32 & STM32CubeIDE Guide

This guide teaches you how to set up STM32CubeIDE, configure the STM32F446RE Nucleo board, and flash firmware to it. This is the toolchain used for all robot firmware.

---

## What is the STM32F446RE?

The STM32F446RE is a microcontroller made by STMicroelectronics. We use it on the **ST Nucleo F446RE** development board, which has an on-board ST-Link debugger -- one USB cable handles both power, flashing, and serial communication.

Key specs:
- 180 MHz ARM Cortex-M4 processor with hardware FPU
- Built-in **bxCAN** peripheral (CAN 2.0B) -- talks to the motors
- USART2 available via ST-Link virtual COM -- talks to your PC
- 512 KB flash, 128 KB RAM
- ST-Link/V2 on-board (no external programmer needed)

All firmware is written in **C** using the **HAL (Hardware Abstraction Layer)** generated by STM32CubeIDE.

---

## What is STM32CubeIDE?

STM32CubeIDE is ST's official IDE for STM32 development. It combines:
- **CubeMX**: a graphical tool for configuring pins and peripherals
- **Eclipse-based editor**: where you write C code
- **ARM GCC toolchain**: compiles your code
- **ST-Link integration**: flashes and debugs over USB

---

## Step 1: Install STM32CubeIDE

1. Go to **st.com/en/development-tools/stm32cubeide.html**
2. Click the Windows installer link
3. Run the installer (it includes all drivers -- accept everything)
4. Launch STM32CubeIDE

This is free. No license needed.

---

## Step 2: Create a new project

1. File -> New -> STM32 Project
2. In the board selector, type `NUCLEO-F446RE` and select it
3. Click Next, give it a name (e.g. `quadruped_firmware`), select C language
4. Click Finish
5. When asked to initialize peripherals to default mode, click **Yes**

CubeMX opens. This is where you configure pins and peripherals.

---

## Step 3: Configure peripherals in CubeMX

### USART2 (serial to PC)
- Left panel: Connectivity -> USART2
- Mode: **Asynchronous**
- Baud rate: **115200**
- Pins PA2 (TX) and PA3 (RX) will be set automatically

### CAN1 (motor bus)
- Left panel: Connectivity -> CAN1
- Activate: check the box
- Prescaler and timing for **1 Mbit/s at 90 MHz APB1:**

| Parameter | Value |
|---|---|
| Prescaler | 9 |
| Time Quanta in Bit Seg 1 | 6 |
| Time Quanta in Bit Seg 2 | 3 |
| ReSynchronization Jump Width | 1 |

- Pins will be set to PA11 (RX) and PA12 (TX) automatically

### NVIC (interrupts)
- Left panel: System Core -> NVIC
- Enable: **USART2 global interrupt**

### Generate code
- Click **Project -> Generate Code** (or the gear icon)
- CubeIDE generates all the HAL init code in `Core/Src/main.c`

---

## Step 4: Add the motor driver files

Copy these two files from the team repo into your CubeIDE project:

- `src/firmware/lib/ak40_driver/ak40_driver.h` -> into `Core/Inc/`
- `src/firmware/lib/ak40_driver/ak40_driver.c` -> into `Core/Src/`

Right-click your project in CubeIDE and refresh so they appear.

---

## Step 5: Add application code to main.c

Open `Core/Src/main.c`. CubeIDE generates `/* USER CODE BEGIN */` and `/* USER CODE END */` comment blocks. **Only add code inside these blocks** -- otherwise CubeMX will delete it when you regenerate.

Reference `src/firmware/src/main.c` in the team repo -- it shows exactly which code goes in which USER CODE section.

Add `#include "ak40_driver.h"` in the Includes section, then add the init calls and command parsing logic as shown in the reference file.

---

## Step 6: Build

Click the **hammer icon** or press Ctrl+B.

A successful build ends with:
```
Build Finished. 0 errors, 0 warnings.
```

---

## Step 7: Flash to the board

1. Plug the Nucleo into your PC with the Mini-B USB cable (the connector nearest the ST-Link chip)
2. Windows installs the ST-Link driver automatically. If not, download from st.com/stsw-link009
3. Click the **green play button** (Run) or press F11
4. CubeIDE flashes and runs the firmware

---

## Step 8: Open the serial monitor

CubeIDE does not have a built-in serial monitor. Use one of these:

- **PuTTY** (free, putty.org): Serial mode, select the correct COM port, 115200 baud
- **Tera Term** (free): Serial -> COM port -> 115200 baud

The COM port number appears in Windows Device Manager under "Ports (COM & LPT)" as "STMicroelectronics Virtual COM Port".

After the STM32 boots you should see:
```
[boot] Dal Robotics Quadruped Firmware
[boot] STM32F446RE  CAN1 PA11/PA12  USART2 PA2/PA3
[ok] CAN bus ready at 1 Mbit/s
[ok] Waiting for commands...
```

## Understanding the code structure

### USER CODE sections

CubeMX regenerates `main.c` every time you change the .ioc config. It preserves everything inside `/* USER CODE BEGIN */` and `/* USER CODE END */` blocks and deletes everything outside. Never put your code outside these blocks.

### The HAL CAN API

```c
// Initialize CAN filter and start the peripheral
AK40_Init(&hcan1);

// Enter MIT mode on motor ID 1
AK40_EnterMode(&hcan1, 1);

// Move motor 1 to 0.5 rad with kp=5, kd=0.5
AK40_State state;
AK40_SendCommand(&hcan1, 1, 0.5f, 0.0f, 5.0f, 0.5f, 0.0f, &state, 10);

// Exit MIT mode
AK40_ExitMode(&hcan1, 1);
```

### The bxCAN peripheral

The STM32F446RE has a built-in CAN 2.0B controller. An external **CAN transceiver** (SN65HVD230 or TJA1050) is required between the STM32 GPIO pins and the physical bus wires (CANH/CANL). The Electrical team provides this circuit.

## Common errors

| Error | Cause | Fix |
|---|---|---|
| ST-Link driver not found | Driver not installed | Download from st.com/stsw-link009 |
| `[error] CAN init failed` | No transceiver or wrong pins | Check wiring to PA11/PA12 |
| Serial monitor shows nothing | Wrong COM port or baud | Check Device Manager for correct port |
| Code deleted after regenerating | Code placed outside USER CODE blocks | Move all code inside the USER CODE sections |
| Build error after adding .c file | File not in build path | Right-click project -> Refresh |
